# Log Analyzer

Профессиональный инструмент для анализа лог-файлов сверхбольшого объема (10GB+). 
Приложение использует механизм Memory Mapping (mmap) для мгновенного доступа к данным без полной загрузки файла в оперативную память.

Разработано для системных администраторов и DevOps-инженеров для быстрого поиска инцидентов и анализа контекста ошибок (строки до/после сбоя).

## Ключевые особенности
*    Открытие файлов любого размера (ограничено только объемом диска, не RAM).
*    Поиск ключевых слов (ERROR, CRITICAL, Exception) с подсветкой синтаксиса.
*    Автоматическое извлечение контекста ошибки (N строк до и после инцидента) для понимания причин сбоя.
*    Сохранение истории поиска и последнего открытого файла (QSettings).
*    Удобная навигация между инцидентами (Next/Prev).

## Технический стек
*    Язык: C++17
*    GUI Фреймворк: Qt 6.4.2 (Widgets)
*    Система сборки: qmake 3.1
*    Библиотеки: Boost.Interprocess (Memory Mapped Files)

## Структура репозитория
*    /OpenLargeFile — Исходный код основного приложения (Qt).
*    FileGenerator — Утилита для генерации тестовых логов.

## Инструкция по сборке и запуску

### Предварительные требования
Необходимо установить библиотеки Boost.
*   **Ubuntu/Debian:** sudo apt install libboost-all-dev
*   **Windows:** Скачать Boost и подключить в .pro файле.

### Сборка
1.  Клонируйте репозиторий: git clone https://github.com/C-plus-plus-Developer/LogAnalyzer.git
2.  Запустите **Qt Creator**.
3.  Нажмите File -> Open File or Project`.
4.  Перейдите в папку OpenLargeFile и выберите файл OpenLargeFile.pro.
5.  Нажмите кнопку "Run" (Зеленый треугольник) или Ctrl+R.

## Тестирование (Генерация логов)
Для проверки производительности в репозитории есть утилита FileGenerator.
1.  Откройте папку FileGenerator.
2.  Скомпилируйте main.cpp любым компилятором (или в IDE).
3.  Запустите полученный exe-файл. Он создаст файл big_log.txt (по умолчанию ~10 млн строк).
4.  Откройте этот файл через Log Analyzer.
